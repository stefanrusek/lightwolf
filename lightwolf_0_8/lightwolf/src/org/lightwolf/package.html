Core classes for flow manipulation and synchronization.

Package Specification

This package contains the {@link org.lightwolf.Flow} class and the {@link org.lightwolf.FlowMethod} annotation, which are used to build flows in Java. A flow is very similar to a thread:
  • Each flow have a stack frame, like a thread.
  • A flow, like a thread, can run Java methods. Such methods can be interpreted but also just-in-time compiled.
  • Flows can access heap objects. The heap space is the same used by threads. Flows and threads can shared such objects in a transparent way.
  • Flows can be suspended or interrupted.
The following summarizes the differences between flows and threads:
  • A flow can be suspended, have its state removed from memory, then restored to memory and then resumed, possibly on another JVM instance or machine.
  • When a flow is suspended, it does not consume a Java thread, and not even a system thread. This makes flows more friendly to thread-pools.
  • The {@link org.lightwolf.Flow} is a final class. To create a flow, one must write and call a {@link org.lightwolf.FlowMethod flow-method}.
